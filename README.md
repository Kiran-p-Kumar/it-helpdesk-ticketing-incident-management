# IT Helpdesk Ticketing & Incident Management System

## Overview
This project simulates a real-world IT helpdesk environment where user issues are tracked, prioritized, resolved, or escalated based on SLA rules. It is designed to understand how technical support teams manage incidents and ensure timely resolution.

## Objectives
- Simulate IT support ticket handling workflow
- Track incidents based on priority and status
- Implement SLA-based resolution logic
- Monitor support performance using dashboards

## Tools Used
- Microsoft Excel
- Pivot Tables
- Data Validation
- Dashboard Design

## Project Features
- 250 simulated IT support tickets
- Issue categories: Login, Network, Software, Hardware
- Priority-based ticket handling (High, Medium, Low)
- Ticket status tracking (Open, In Progress, Resolved, Escalated)
- SLA rules with resolution time logic
- Support performance dashboard

## Key Metrics Tracked
- Total Tickets
- Resolved Tickets
- Escalated Tickets
- Average Resolution Time
- SLA Compliance
- Tickets by Priority
- Tickets by Issue Type

## Workflow
1. Ticket creation with issue type and priority
2. Assigning status based on progress
3. Applying SLA rules for resolution time
4. Escalating unresolved or delayed tickets
5. Monitoring performance through dashboard

## Files Included
- IT_Helpdesk_Ticketing_Incident_Management_Project.xlsx
- Dashboard screenshots
- SLA rules sheet

## Learning Outcome
This project helped in understanding IT support operations, incident management, ticket prioritization, SLA tracking, escalation handling, and performance monitoring in a structured support environment.

## Note
This is a simulated project built based on common real-world IT support scenarios to understand helpdesk workflow and service operations.
